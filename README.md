# Backlist เช็คก่อนโอน

เว็บแอปสำหรับเก็บรายชื่อบุคคล/บัญชี/เพจที่ถูกแจ้งว่ามีพฤติกรรมฉ้อโกง เพื่อให้คนอื่นตรวจสอบก่อนซื้อขาย

## โครงสร้างไฟล์

```
├── index.html      หน้าเว็บหลัก (โครงสร้าง HTML)
├── style.css       สไตล์ทั้งหมด (โทนมิ้นต์น่ารัก)
├── app.js          ตัวควบคุมหลัก (เริ่มแอป, ค้นหา, กรอง)
├── storage.js      จัดการข้อมูลผ่าน localStorage (CRUD)
├── renderer.js     สร้าง DOM การ์ดรายชื่อ
├── modal.js        จัดการ modal เพิ่ม/แก้ไขข้อมูล
├── toast.js        ระบบแจ้งเตือนแบบ toast
├── utils.js        ฟังก์ชันช่วยทั่วไป (format วันที่, escape HTML ฯลฯ)
└── README.md       ไฟล์นี้
```

## วิธีใช้งาน

1. เปิดไฟล์ `index.html` ในเบราว์เซอร์ (ดับเบิลคลิกได้เลย ไม่ต้องมีเซิร์ฟเวอร์)
2. กด "แจ้งแบล็กลิสต์" เพื่อเพิ่มรายชื่อใหม่
3. ใช้ช่องค้นหาหรือปุ่มกรองตามประเภทเพื่อค้นหารายชื่อ
4. กด "แก้ไข" หรือ "ลบ" บนการ์ดเพื่อจัดการรายชื่อ
5. ใช้ปุ่ม "ส่งออกข้อมูล" เพื่อบันทึกข้อมูลเป็นไฟล์ .json สำรอง และ "นำเข้าข้อมูล" เพื่อโหลดกลับ

## ข้อมูลเก็บที่ไหน?

ข้อมูลทั้งหมดเก็บไว้ใน `localStorage` ของเบราว์เซอร์เครื่องนั้นๆ เท่านั้น
- ✅ ไม่ต้องมีเซิร์ฟเวอร์ ไม่ต้องเสียค่าใช้จ่าย
- ⚠️ ข้อมูลจะไม่ sync ข้ามเครื่อง/เบราว์เซอร์ และจะหายถ้าล้างข้อมูลเบราว์เซอร์
- หากต้องการให้หลายคนเข้าถึงข้อมูลชุดเดียวกัน (เช่น เก็บใน Google Sheet หรือฐานข้อมูลกลาง) จะต้องเพิ่มระบบ backend แยกต่างหาก

## การปรับแต่ง

- เปลี่ยนสีธีม: แก้ค่าตัวแปรใน `:root` ที่ด้านบนของ `style.css`
- เพิ่ม/ลดประเภทการโกง: แก้ไข `<select id="inputTag">` ใน `index.html` และ array `TAGS` ใน `app.js`, รวมถึงเพิ่ม class `.tag-xxx` ใน `style.css` และ mapping ใน `utils.js` (`tagClass`)

## ข้อควรระวัง (สำคัญ)

เว็บประเภทนี้มีความเสี่ยงด้านกฎหมาย หากเผยแพร่สู่สาธารณะ:
- การกล่าวหาว่าใครเป็น "คนโกง" โดยไม่มีคำพิพากษา อาจเข้าข่ายหมิ่นประมาทได้ ถึงแม้ข้อมูลจะเป็นจริง
- ควรปกปิดข้อมูลส่วนตัว (เลขบัญชี/เบอร์โทรเต็ม) ให้เหลือเพียงบางส่วน เช่น xxx-x-x4567-x
- ควรมีระบบยืนยันหลักฐาน และช่องทางให้ผู้ถูกแจ้งชี้แจง/ขอลบข้อมูล
- แนะนำให้ปรึกษาผู้เชี่ยวชาญด้านกฎหมายก่อนเปิดให้บุคคลทั่วไปใช้งานจริง
